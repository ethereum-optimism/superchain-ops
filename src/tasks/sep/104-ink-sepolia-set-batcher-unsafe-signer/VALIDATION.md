# Validation

This document can be used to validate the inputs and result of the execution of the transaction which you are signing.

The steps are:

1. [Validate the Domain and Message Hashes](#expected-domain-and-message-hashes)
2. [Transaction Inputs](config.toml): inputs can be verified in the config.toml file.
3. State Changes: the template's `_validate` block asserts `SystemConfig.batcherHash()` and `SystemConfig.unsafeBlockSigner()` equal the configured values. State changes can also be reviewed in Tenderly via the link printed during simulation.

## Expected Domain and Message Hashes

First, validate the domain and message hashes. These values should match both the values on your ledger and the values printed to the terminal when you run the task.

> [!CAUTION]
>
> Before signing, ensure the below hashes match what is on your ledger.
>
> ### OPE Safe (`0x837DE453AD5F21E89771e3c06239d8236c0EFd5E`)
>
> - Domain Hash:  `0xe84ad8db37faa1651b140c17c70e4c48eaa47a635e0db097ddf4ce1cc14b9ecb`
> - Message Hash: `0x4fb9844cc9182603b091a1ef7341ed62b052c60de6c7a8da700df29e81ea6f22`
> - Safe Hash:    `0x6473018a873abe8d25c46162b3d7b7734ce9bcec8f4475fbc5c137dbfc269843`
>
> _The Domain Hash is deterministic for the OPE Safe on Sepolia (chainId 11155111): `keccak256(abi.encode(0x47e7…9218, 11155111, 0x837DE4…Fd5E))`. The Message and Safe hashes above were generated by a successful `just simulate` run with the [config.toml](./config.toml) state overrides (**OPE Safe nonce = 12**; SystemConfig.owner → OPE Safe). They are valid only for those overrides — re-run `just simulate` and replace them if the OPE Safe nonce has advanced or the owner override is removed (after W26) before signing._

## Understanding Task Calldata

The task batches two `SystemConfig` setters through Multicall3, targeting the Ink Sepolia `SystemConfigProxy` (`0x05C993e60179f28bF649a2Bb5b00b5F4283bD525`).

Verify the two inner calldata fingerprints:

```bash
# setBatcherHash(bytes32) — OPE batcher address left-padded to 32 bytes
cast calldata "setBatcherHash(bytes32)" 0x00000000000000000000000042c4dbdd6c573b9c2a8657f59ee8e84ae45f2695
# Expected: 0xc9b26f6100000000000000000000000042c4dbdd6c573b9c2a8657f59ee8e84ae45f2695

# setUnsafeBlockSigner(address) — OPE sequencer
cast calldata "setUnsafeBlockSigner(address)" 0x900C36E0CcAC26F9F49ECa6bbD748e347531864b
# Expected: 0x18d13918000000000000000000000000900c36e0ccac26f9f49eca6bbd748e347531864b
```

The outer Multicall3 `aggregate3` blob is assembled by the framework and printed during simulation; confirm it contains both inner calls above and no others.

## Task State Changes

### `0x05C993e60179f28bF649a2Bb5b00b5F4283bD525` (SystemConfigProxy) — Chain ID 763373

- `batcherHash()` updates from `0x000…21e57c21530bc33f12ba96c9ddc135488365002f` (Gelato) to `0x000…42c4dbdd6c573b9c2a8657f59ee8e84ae45f2695` (OPE batcher).
- `unsafeBlockSigner()` updates from `0x43ec5732581d3FAE18AbB7CE34a796E111dBD1a0` (Gelato) to `0x900C36E0CcAC26F9F49ECa6bbD748e347531864b` (OPE sequencer).

### `0x837DE453AD5F21E89771e3c06239d8236c0EFd5E` (OPE Safe)

Nonce increments by 1.

> [!NOTE]
> The `SystemConfig.owner()` change Gelato → OPE Safe (slot `0x33`) is **not** part of this task; it is performed outside the ops repo by Gelato (plan step W26). The owner override in [config.toml](./config.toml) exists only so the setters authorize during simulation, and must be removed once W26 is executed on-chain.

## Post-execution verification

```bash
cast call 0x05C993e60179f28bF649a2Bb5b00b5F4283bD525 "batcherHash()(bytes32)" --rpc-url <SEPOLIA_RPC>
# Expected: 0x00000000000000000000000042c4dbdd6c573b9c2a8657f59ee8e84ae45f2695
cast call 0x05C993e60179f28bF649a2Bb5b00b5F4283bD525 "unsafeBlockSigner()(address)" --rpc-url <SEPOLIA_RPC>
# Expected: 0x900C36E0CcAC26F9F49ECa6bbD748e347531864b
```
